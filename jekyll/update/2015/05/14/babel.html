Stay Ahead of the Curve. Start Using ES6 On All Your Projects Using Babel.

May 14, 2015

Get Ready for ES6… By Using It Now

ECMAScript 6 scheduled officially to roll out in June 2015, but it’s been available for use for a while now. Not only that, but you can easily use a transpiler, like Babel or Traceur, to convert all ES6 code into JavaScript to maintain compatibility with current browsers.

Using Babel In Two Steps

Babel is shockingly easy to use, and the code it transpiles looks good, as though a reasonable human wrote it.

It takes only two shell commands. First, install install Babel in the command line using npm:

$ npm install -g babel

Then, to convert any file you write using ES6, use the babel command with (a) the filename, (b) the flag --out-file and (c) an output filename:

$ babel es6-script.js --out-file script-compiled.js

Alternatively, you could transpile an entire directory of files into another by using the --out-dir flag:

$ babel src --out-dir lib

Or, even cooler, transpile the entire src directory and output it to the one concatenated file:

$ babel src --out-file script-compiled.js

Add Babel To Your Build Process For Automatic Transpiling

So that you don’t have to do this with every time you make edits to any file, add Babel to your build process. Gulp has a couple great options that have been easy to implement: gulp-babel and babelify, for those who also love using Browserify (especially with React).

Use Source Maps To Help In Debugging

I’ve worked on projects in CoffeeScript before and realize how annoying it can be to debug an application when errors continually occur in files that you’re not working on. The line numbers are all different, and the code feels foreign to you.

That’s what source maps are for. They tell you which line in the original files that the output code came from, whether transpiled, minified, or concatenated. You can also see that they’re implemented in the example Gulpfile shared above. For more, here is a clear and practical post on starting to use sourcemaps.

Adopting ES6 now is a great investment in yourself. Tools like Babel, gulp-babel/babelify, and source maps make it easy to stay ahead of the curce.
